## musl vs gilbc

most of the world's C compilations for linux is in gilbc. Windows has it's own compilation protocols..

```bash
pip install numpy
```

Try running this in musl void linux, and you get an error of pip being externally managed and to create a venv virtual environments.

My point is, the void linux OS I use, is musl version. Only binaries compiled against musl can be executed here. And **honestly**, it allows the OS to be lightweight.

Only problem here is that you can't install proprietary apps and packages that are compiled to run on main OS distributions: ubuntu, windows, macOS.
